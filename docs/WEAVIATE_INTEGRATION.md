# Weaviate Integration Guide

This document explains the Weaviate vector database integration for the Document AI chatbot.

## Overview

The system now supports **dual vector storage**:
- **FAISS** (local file-based index) - existing functionality
- **Weaviate Cloud** (cloud-based vector database) - new functionality

When you upload a PDF, the document chunks are automatically:
1. Extracted via OCR
2. Split into manageable chunks
3. Embedded using OpenAI's text-embedding-ada-002
4. Stored in both FAISS (local) and Weaviate (cloud)

## Environment Variables

Make sure these are set in your `.env` file:

```env
# OpenAI Configuration
OPENAI_API_KEY=your_openai_api_key_here

# Weaviate Cloud Configuration
WEAVIATE_URL=https://your-cluster.weaviate.cloud
WEAVIATE_ADMIN_KEY=your_weaviate_admin_key_here
WEAVIATE_VIEW_KEY=your_weaviate_view_key_here

# Collection Name (optional, defaults to LegalKnowledgeBase)
WEAVIATE_COLLECTION=LegalKnowledgeBase
```

## Usage

### 1. Uploading Documents (Automatic Upsertion)

When you upload a PDF via the existing endpoint, it automatically upserts to both FAISS and Weaviate:

```bash
curl -X POST "http://127.0.0.1:8000/upload_legal_pdf" \
  -F "file=@your_document.pdf"
```

**Response:**
```json
{
  "message": "PDF 'your_document.pdf' processed successfully",
  "chunks": 42
}
```

### 2. Searching Weaviate

Search for similar documents in the Weaviate collection:

```bash
curl -X POST "http://127.0.0.1:8000/weaviate_search" \
  -F "query=What are the contract terms?" \
  -F "k=5"
```

**Optional: Filter by filename**
```bash
curl -X POST "http://127.0.0.1:8000/weaviate_search" \
  -F "query=What are the contract terms?" \
  -F "k=5" \
  -F "filename=contract.pdf"
```

**Response:**
```json
{
  "status": "success",
  "query": "What are the contract terms?",
  "results_count": 5,
  "results": [
    {
      "content": "The contract terms specify...",
      "filename": "contract.pdf",
      "chunk_id": 3,
      "distance": 0.234,
      "metadata": "{\"pdf_path\": \"/tmp/xyz.pdf\"}"
    }
  ]
}
```

### 3. Getting Collection Statistics

Check how many documents are stored in Weaviate:

```bash
curl -X GET "http://127.0.0.1:8000/weaviate_stats"
```

**Response:**
```json
{
  "collection_name": "LegalKnowledgeBase",
  "total_objects": 127,
  "status": "active"
}
```

### 4. Deleting Documents

Remove all chunks associated with a specific filename:

```bash
curl -X POST "http://127.0.0.1:8000/weaviate_delete" \
  -F "filename=contract.pdf"
```

**Response:**
```json
{
  "status": "success",
  "filename": "contract.pdf",
  "deleted_count": 42
}
```

## Programmatic Usage

### Direct Upsertion

You can also use the Weaviate functions directly in your Python code:

```python
from RAG.weaviate_vectorizer import upsert_to_weaviate

chunks = ["chunk 1 text", "chunk 2 text", "chunk 3 text"]
result = upsert_to_weaviate(
    chunks=chunks,
    filename="my_document.pdf",
    metadata={"source": "upload", "date": "2025-10-23"}
)

print(result)
# Output: {
#   "status": "success",
#   "total_chunks": 3,
#   "successful_insertions": 3,
#   "failed_insertions": 0,
#   "filename": "my_document.pdf"
# }
```

### Direct Search

```python
from RAG.weaviate_vectorizer import search_weaviate

results = search_weaviate(
    query="What are the contract terms?",
    k=5,
    filename="contract.pdf"  # optional filter
)

for result in results:
    print(f"Content: {result['content']}")
    print(f"Distance: {result['distance']}")
```

### Using the WeaviateVectorStore Class

For more control, use the class directly:

```python
from RAG.weaviate_vectorizer import WeaviateVectorStore

store = WeaviateVectorStore()
try:
    # Upsert documents
    result = store.upsert_document_chunks(
        chunks=["chunk 1", "chunk 2"],
        filename="document.pdf",
        metadata={"author": "John Doe"}
    )
    
    # Search
    results = store.search_similar("contract terms", k=5)
    
    # Get stats
    stats = store.get_collection_stats()
    
    # Delete by filename
    delete_result = store.delete_by_filename("old_document.pdf")
    
finally:
    store.close()
```

## Architecture

### Collection Schema

The Weaviate collection has the following schema:

- **content** (TEXT): The text content of the document chunk
- **filename** (TEXT): The original filename of the document
- **chunk_id** (INT): The chunk index within the document
- **metadata** (TEXT): Additional metadata in JSON format
- **vector**: Automatically generated by Weaviate's text2vec-openai vectorizer

### Vectorization

- Uses OpenAI's `text-embedding-ada-002` model
- Embeddings are generated automatically by Weaviate
- Supports semantic search with vector similarity

## Features

✅ **Automatic Upsertion**: Documents are automatically uploaded to Weaviate when PDFs are processed  
✅ **Dual Storage**: Maintains both FAISS (local) and Weaviate (cloud) indexes  
✅ **Semantic Search**: Vector similarity search for finding relevant document chunks  
✅ **Metadata Support**: Store additional metadata with each chunk  
✅ **Batch Operations**: Efficient batch insertion of multiple chunks  
✅ **Error Handling**: Graceful fallback if Weaviate fails (FAISS continues to work)  
✅ **Filtering**: Filter search results by filename  
✅ **Collection Management**: View stats and delete documents  

## Troubleshooting

### Connection Issues

If you see connection errors:
1. Verify your `WEAVIATE_URL` is correct
2. Check that your `WEAVIATE_ADMIN_KEY` is valid
3. Ensure your Weaviate cluster is active

### Authentication Errors

If authentication fails:
- Double-check the API key format
- Ensure you're using the admin key (not the view key) for write operations

### Missing Environment Variables

If you see `ValueError: WEAVIATE_URL and WEAVIATE_ADMIN_KEY must be set`:
- Make sure your `.env` file exists
- Verify the environment variables are properly loaded
- Restart your application after updating `.env`

## Best Practices

1. **Use meaningful filenames**: The filename is used as a key identifier for filtering and deletion
2. **Include metadata**: Store additional context in the metadata field for better tracking
3. **Monitor collection size**: Use `/weaviate_stats` to track growth
4. **Clean up old documents**: Delete outdated documents to keep the collection relevant
5. **Handle errors gracefully**: The system will continue with FAISS if Weaviate fails
